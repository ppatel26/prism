# Archetypes (the skeleton)

An archetype is what a single content block *expresses* - classified by the **data relationship**, not by appearance. Classifying by function keeps the set finite (there are only so many relationships); classifying by appearance does not (see Lengler & Eppler's Periodic Table of Visualization Methods, which hit ~100 cells and still is not complete).

This axis seeds from the Financial Times "Visual Vocabulary" nine functional families:

| Archetype | Relationship it expresses |
|-----------|---------------------------|
| deviation | variation from a reference / baseline |
| correlation | relationship between 2+ variables |
| ranking | order / position |
| distribution | spread and frequency |
| change-over-time | trends, cycles (timeline) |
| part-to-whole | components of a total (layered stack) |
| magnitude | size comparisons (matrix / comparison) |
| spatial | location / geography |
| flow | movement between states (pipeline) |

Composites (e.g. "pipeline + matrix + timeline") are NOT new archetypes - they are multiple atomic blocks arranged by a `layout`. Keep archetypes atomic; let layout do the combining.

**References:** FT Visual Vocabulary; Wilkinson, *The Grammar of Graphics*; Dan Roam, *The Back of the Napkin* (six questions -> six frames); Bertin, *Semiology of Graphics*.
