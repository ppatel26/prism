# Layouts (the arrangement)

A layout decides WHERE blocks go - independent of what they contain (archetype) or how they look (style). It is the combinator that turns a finite set of atomic archetype blocks into effectively infinite posters.

Four sub-parts keep non-linear, organic layouts possible while staying finite:

## topologies/ (the macro-shape of the page)

- `grid` - tidy rows / columns
- `radial` - hub-and-spoke
- `network` - node-link (non-linear)
- `layered` - overlap / z-depth
- `freeform` - organic canvas

(An extension of Bertin's diagrams / networks / maps.)

## operators/ (composition grammar - how blocks nest)

- `nest`, `layer`, `concat`, `facet`, `repeat`

(Borrowed from Vega-Lite's view composition. Recursion = a slot can hold a whole sub-layout.)

## edges/ (the connection layer drawn on top of placement)

- `arrow`, `thread`, `annotation`, `flow`

(Where the creative non-linearity lives - mind maps, causal loops, subway maps.)

## constraints/ (the escape hatch from fixed cells)

- `focal-point`, `align`, `proximity`, `balance`

(Declare relationships, let a solver place organically. Cf. the Cassowary solver behind Apple Auto Layout.)
